# 🩺 Diary of Symptoms: Ваш Персональный AI-Аналитик Здоровья

[!Python](https://www.python.org/)
[!FastAPI](https://fastapi.tiangolo.com/)
[!PostgreSQL](https://www.postgresql.org/)
[!Google Gemini](https://ai.google.dev/models/gemini)
[!License: MIT](https://opensource.org/licenses/MIT)

Добро пожаловать в **Diary of Symptoms** — инновационное REST API для ведения дневника симптомов, обогащенное мощным AI-анализом от Google Gemini. Этот проект создан, чтобы помочь пользователям отслеживать свое самочувствие, фиксировать симптомы, прием лекарств и получать персонализированные инсайты и рекомендации на основе введенных данных.

---

## ✨ Ключевые Особенности

*   **Интеллектуальный AI-Анализ**: Автоматическая генерация инсайтов и потенциальных триггеров для ваших симптомов с помощью Google Gemini 2.0 Flash.
*   **Детальное Отслеживание Симптомов**: Фиксируйте тяжесть, длительность, сопутствующие факторы (сон, стресс, питание) и заметки.
*   **Управление Лекарствами**: Добавляйте и отслеживайте принимаемые медикаменты, аллергии и диагнозы.
*   **Безопасная Аутентификация**: Регистрация и авторизация пользователей для защиты ваших личных данных.
*   **Гибкая API-Архитектура**: Чистый и хорошо документированный REST API, построенный на FastAPI.
*   **Надежное Хранение Данных**: Использование PostgreSQL для стабильного и масштабируемого хранения информации.

---

## 🛠 Стек Технологий

*   **Backend**: FastAPI (Python) + Uvicorn
*   **Database**: PostgreSQL с SQLAlchemy ORM
*   **AI**: Google Gemini 2.0 Flash
*   **Валидация Данных**: Pydantic
*   **Управление Зависимостями**: Poetry (рекомендуется) или `pip`
*   **Переменные Окружения**: python-dotenv
*   **Генерация PDF (потенциально)**: WeasyPrint (зависимость, не используется напрямую в текущем API, но может быть расширением)

## 🚀 Установка и Запуск

Следуйте этим шагам, чтобы развернуть проект локально.

### 1. Клонирование репозитория

```bash
git clone https://github.com/Nikola/diary-of-symptoms.git # Замените на актуальный URL репозитория
cd diary-of-symptoms
```

### 2. Создание виртуального окружения и установка зависимостей

Рекомендуется использовать `Poetry` для управления зависимостями:

```bash
# Установка Poetry (если еще не установлен)
pip install poetry

# Установка зависимостей проекта
poetry install
```

Или с помощью `pip`:

```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения

Создайте файл `.env` в корневой директории проекта, скопировав `.env.example`, и заполните его своими значениями:

```bash
cp .env.example .env
```

Пример содержимого `.env`:

```
DATABASE_URL="postgresql://user:password@host:5432/database_name"
CORS_ORIGINS="http://localhost:5173,http://localhost:3000"
GEMINI_API_KEY="ВАШ_API_КЛЮЧ_GOOGLE_GEMINI"
APP_NAME="Diary Of Symptoms API"
SECRET_KEY="your_super_secret_jwt_key" # Сгенерируйте случайный и надежный ключ
ALGORITHM="HS256"
ACCESS_TOKEN_EXPIRE_MINUTES="30"
```
**Важно**: Получите ваш `GEMINI_API_KEY` на Google AI Studio.

### 4. Запуск сервера

```bash
# Если используете Poetry
poetry run uvicorn app.services.main:app --host 0.0.0.0 --port 8000 --reload

# Если используете pip
cd backend
uvicorn app.services.main:app --host 0.0.0.0 --port 8000 --reload
```

Сервер будет доступен по адресу `http://localhost:8000`. Документация API (Swagger UI) будет доступна по адресу `http://localhost:8000/docs`.

## 🔌 API Endpoints

Базовый URL: `/api`

### 👤 Пользователи — `/api/users`

| Метод | Endpoint | Описание |
| :---- | :------- | :--------- |
| `POST` | `/register` | Регистрация нового пользователя (201 Created) |
| `POST` | `/login` | Авторизация пользователя |
| `GET` | `/{user_id}` | Получение пользователя по ID |

### 💊 Лекарства — `/api/medications`

| Метод | Endpoint | Описание |
| :---- | :------- | :--------- |
| `POST` | `/add` | Добавление лекарства (201 Created) |
| `GET` | `/{medication_id}` | Получение лекарства по ID |

### 📋 Записи Симптомов — `/api/symptom-entries`

| Метод | Endpoint | Описание |
| :---- | :------- | :--------- |
| `POST` | `/add` | Создание записи симптома с AI-анализом (201 Created) |
| `GET` | `/{entry_id}` | Получение записи симптома по ID |

## ⚙️ Конфигурация

Приложение загружает настройки из `.env` файла. Убедитесь, что все необходимые переменные установлены:

| Переменная | Описание | Пример |
| :--------- | :------- | :----- |
| `DATABASE_URL` | Строка подключения к базе данных PostgreSQL. | `postgresql://user:pass@localhost:5432/mydb` |
| `CORS_ORIGINS` | Разрешённые домены для CORS (через запятую). | `http://localhost:5173,http://localhost:3000` |
| `GEMINI_API_KEY` | **Обязательно.** Ваш API-ключ для Google Gemini. | `AIzaSyC...` |
| `APP_NAME` | Название приложения. | `"Diary Of Symptoms API"` |
| `SECRET_KEY` | **Обязательно.** Секретный ключ для JWT токенов (генерируйте случайный). | `your_super_secret_jwt_key` |
| `ALGORITHM` | Алгоритм хеширования для JWT. | `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Время жизни access токена в минутах. | `30` |

## 🧩 Модели Данных

### `User`
*   `id`: Уникальный идентификатор пользователя.
*   `name`: Имя пользователя.
*   `email`: Электронная почта (уникальная).
*   `hashed_password`: Хеш пароля пользователя.
*   `google_id`: ID пользователя Google (для будущей интеграции).
*   `email_verified`: Статус верификации почты.
*   `plan_type`: Тип плана (например, "free", "premium").
*   `weight`: Вес пользователя.
*   `height`: Рост пользователя.
*   `created_at`: Дата и время создания записи.

### `Medication`
*   `id`: Уникальный идентификатор лекарства.
*   `regular_medications_dosage`: Дозировка регулярных лекарств.
*   `notes`: Дополнительные заметки.
*   `created_at`: Дата и время создания записи.
*   `allergy`: Список аллергий.
*   `diagnosis`: Диагноз.
*   `user_id`: Внешний ключ к `User`.

### `SymptomEntry`
*   `id`: Уникальный идентификатор записи симптома.
*   `user_id`: Внешний ключ к `User`.
*   `start_at`: Дата и время начала симптома.
*   `symptom`: Описание симптома.
*   `severity`: Тяжесть симптома (от 0 до 10).
*   `duration`: Длительность симптома.
*   `notes`: Дополнительные заметки.
*   `sleep_quality`: Качество сна (от 0 до 10).
*   `sleep_hours`: Количество часов сна.
*   `stress_level`: Уровень стресса (от 0 до 10).
*   `food_notes`: Заметки о питании.
*   `medications_taken`: Принятые лекарства (текстовое поле).
*   `ai_insights`: AI-анализ и рекомендации.
*   `created_at`: Дата и время создания записи.

## ⚠️ Известные Проблемы и Планы на Будущее

Мы постоянно работаем над улучшением проекта. Вот некоторые известные моменты и планы:

*   **Безопасность Паролей**: Текущее хеширование SHA-256 будет заменено на более надежные алгоритмы, такие как `bcrypt` или `argon2`, для повышения безопасности.
*   **Frontend**: Директория `frontend/` пока пуста. В планах разработка интуитивно понятного пользовательского интерфейса для взаимодействия с API.
*   **Импорты Моделей**: Необходимо исправить несоответствие имен файлов и импортов в `backend/app/models/__init__.py` для лучшей организации кода.
*   **Тестирование**: Добавление модульных и интеграционных тестов для обеспечения стабильности и надежности.
*   **Дополнительные AI-Функции**: Расширение возможностей AI для более глубокого анализа и прогнозирования.
*   **Уведомления**: Реализация системы уведомлений для напоминаний о приеме лекарств или регистрации симптомов.
